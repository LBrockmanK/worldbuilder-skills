# Brief Procedure

Follow this procedure before dispatching the trial. It produces `brief.md`, the frozen input the trial works from.

## 1. Pick the character

Pick one pending character from the project's cast plan. Note whether the project plan flags this character for intimate dynamics. Keep this flag state on hand: it becomes a single line in `brief.md`.

## 2. Run the Q&A

Run the project's normal character Q&A session with the user. Follow the project's installed character skill exactly as it prescribes. Stop once the session notes capture is done. Do not write any note sections for this character.

Scope note: elsewhere in this kit, the installed-skill ban applies only to the six writing agents that later draft the character note. It does not apply here. Using the project's installed character skill for this Q&A is expected and required.

## 3. Freeze the brief

Copy the captured session notes transcript verbatim into `brief.md` in the kit folder. Prefix the transcript with two lines, in this order:

1. The character's name.
2. The intimate-dynamics flag state (flagged or not flagged).

## 4. Treat the brief as read-only

Once `brief.md` is frozen, it does not change for the rest of this run. If the user volunteers more detail about the character later, do not add it to `brief.md`. That detail is out of scope for this run.
