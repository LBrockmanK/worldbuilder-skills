---
name: feedback-character-initial-state
description: Character notes describe the character as they are on their available day only — no forward references outside Storylines
metadata:
  type: feedback
---

Character notes describe each character's starting state as of the day they are introduced. Nothing outside the Storylines section may reference events that haven't happened yet or developments that require time to unfold.

Example: if characters A, B, and C are all available Day 1, and a canon event involving all three occurs on Day 5, none of their notes may reference that event except in Storylines. A character available from Day 5 onwards may reference the event because it has already occurred when they are introduced.

The rule: if a character is introduced before an event, that event does not exist in their note outside Storylines. Even "may happen someday" framing belongs in Storylines, not in the main note body.

**Why:** Character notes are starting points for an LLM-driven world. Forward references in the main body contaminate the character's initial presentation and break the living-world illusion.

**How to apply:** When writing or reviewing any section other than Storylines, check the character's available day and ask: has this happened yet? If not, move it or cut it.
